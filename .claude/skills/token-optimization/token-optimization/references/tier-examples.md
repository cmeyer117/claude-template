# Tier Classification Examples

Use these as calibration references when classifying task size.

---

## S — Small (Just Do It)

- "Fix this bug in my function" (1 function)
- "What does `useEffect` do?"
- "Write a regex to match email addresses"
- "What's the difference between `==` and `===`?"
- "Add error handling to this snippet"
- "Rename this variable everywhere in this file"
- "Quick explanation of how JWT works"
- "What CPA courses do I still need?"
- "Translate this SQL query to TypeScript"

---

## M — Medium (Do It, Stay Tight)

- "Build a custom React hook for form validation"
- "Write a Node.js middleware for auth"
- "Explain the full JWT auth flow with code"
- "Draft a brand pitch email for a streamer"
- "Write a multi-step n8n workflow description"
- "Explain compound vs isolation exercises for bodybuilding"
- "Set up a basic Express API with 3 routes"
- "Create a TypeScript interface schema for my accounting system"
- "Write a Vapi system prompt for my AI receptionist"

---

## L — Large (⚠️ Checkpoint First)

- "Build a full React component with state, API calls, and UI"
- "Create a complete pitch deck outline with all sections"
- "Write the full auth module for my app (JWT + refresh tokens)"
- "Generate a weekly meal + training plan for contest prep"
- "Design the full database schema for my accounting firm software"
- "Write a detailed business plan section"
- "Build a multi-step form with validation in React"
- "Create a full n8n workflow for client onboarding"
- "Write a 10-email drip sequence for brand outreach"

---

## XL — Massive (🚨 Must Checkpoint + Plan First)

- "Scaffold the entire Jarvis multi-agent system"
- "Build the full accounting firm automation backend"
- "Create a complete React coaching dashboard (all components)"
- "Write all the API endpoints for my SaaS"
- "Generate a full competitive bodybuilding contest prep plan (16 weeks)"
- "Build a complete brand sponsorship deck with all streamer data"
- "Create the entire CPA study curriculum with timelines"
- "Rewrite my entire codebase with TypeScript strict mode"
- "Design the full system architecture + ERD + API spec for my firm"

---

## Special Cases

### Iterative Rewrites
If the user asks to "refactor," "rewrite," or "clean up" a large existing file — classify one tier HIGHER than the original task, because you'll need to show before/after context.

### Debugging Full Codebases
Always checkpoint. Offer to: (a) review just the error + stack trace, (b) audit one module at a time, (c) list likely suspects only.

### Multi-File Outputs
Any task requiring 3+ files = at minimum L, often XL. Checkpoint.
